# 个人笔记网页Znotes

完全由ai agents开发


源于本人一直缺一个markdown笔记管理搜索系统（虽然出来后没有typora用的顺。。。但至少有了搜索功能）

其实似乎还不错？

下面介绍也由ai写，但是检查后没有错误


真的可以试试！


这是一个基于 Flask 的个人笔记网页应用，自动从 `notes` 目录扫描 Markdown 文件，并按文件夹结构生成可展开的侧边栏树形导航。

## 主要功能

- 自动发现 `notes/` 下的所有 `.md` 文件
- 左侧展示目录树，支持展开/收起文件夹
- 渲染 Markdown，并使用 KaTeX 自动渲染 `$...$` 和 `$$...$$` 公式
- 即时搜索：输入关键词即可快速搜索文件名、路径和内容
- 搜索结果显示匹配来源，支持“打开内容”和“打开文件夹”操作
- 容错处理长代码块，自动换行并支持横向滚动

## 安装与运行

1. 创建并激活 Python 虚拟环境（可选）
   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   ```
2. 安装依赖
   ```bash
   pip install flask markdown2 jieba
   ```
3. 启动应用
   ```bash
   python app.py
   ```
4. 在浏览器中打开
   ```text
   http://127.0.0.1:5000
   ```

## 使用说明

- 将笔记文件放入 `notes` 目录，支持多级子目录
- 每个 `.md` 文件会作为一个独立笔记显示，文件名作为标题
- 左侧侧栏会按文件夹结构显示所有笔记
- 点击笔记名称可查看内容
- 搜索框可实时查询笔记，搜索结果会显示匹配类型
- 若搜索结果来自文件夹，可直接展开该文件夹

## Notes 目录示例

```
notes/
  个人计划/
    2026-目标.md
    读书笔记.md
  技术/
    Python.md
    Flask.md
  README.md
```

## 公式支持

- 支持行内公式：`$a^2 + b^2 = c^2$`
- 支持块公式：
  ```md
  $$
  E = mc^2
  $$
  ```

## 常见问题

- `打开文件夹` 按钮只要该笔记位于子目录中就会显示
- 根目录笔记不显示“打开文件夹”按钮，因为它本身没有父文件夹
- 如果搜索没有结果，请确认关键词是否存在于文件名、路径或内容中

## 其他说明

- 如果你想重新扫描笔记目录，可访问 `/reindex`
- 静态资源在 `static/` 中，页面模板在 `templates/` 中
- 可根据需要继续扩展编辑、标签、主题等功能
